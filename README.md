# test-openspec-todolist

A minimal ToDoList application built as a testbed for **OpenSpec** (Spec-Driven Development).

This repository is intentionally structured for maximum compatibility with modern AI coding agents while maintaining clean industry standards.

### Root Documentation

- **`AGENTS.md`** — Pure agentic rules, behavior, and skills for AI agents
- **`CONVENTIONS.md`** — Project conventions, code style, contribution guidelines, and permanent tech stack
- **`GLOSSARY.md`** — Project terminology and definitions
- **`README.md`** — This file

### Purpose

- Test and refine OpenSpec workflow in real conditions
- Keep industry-wide artifacts at the root level
- Maintain clear separation between agnostic standards and OpenSpec-specific logic (`openspec/` directory)

### Quick Start

1. Clone the repository
2. Open in VSCode + Continue.dev
3. Read `AGENTS.md` first (agent behavior)
4. Follow `CONVENTIONS.md` for all development rules and tech constraints

All development must respect the OpenSpec process.
