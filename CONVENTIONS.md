# CONVENTIONS.md

## Project Conventions & Contribution Guidelines

This is a test ToDoList project developed with OpenSpec (Spec-Driven Development).

### General Conventions
- All documentation is in English.
- Use Markdown for all artifacts.
- Follow OpenSpec for internal process.
- Industry-wide rules live at repository root.
- OpenSpec-specific rules live under `openspec/`.

### Contribution Flow
1. Read root industry files first.
2. Check `openspec/config.yaml` for ownership and references.
3. For changes to industry files: edit directly (they are masters).
4. For OpenSpec-specific logic: work inside `openspec/specs/`.

### Naming & Structure
- Use kebab-case for directories and files when possible.
- Prefer conventional root locations (AGENTS.md, CODE_STYLE.md, etc.).

See also: CODE_STYLE.md, GLOSSARY.md, SKILLS.md
