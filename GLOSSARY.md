# GLOSSARY.md

## Project Glossary

### Core Terms

- **OpenSpec**: Spec-Driven Development methodology used as the internal process framework for this ToDoList project.
- **Industry-wide artifacts**: Files at repository root (AGENTS.md, CONVENTIONS.md, etc.) that are AI-vendor and technology agnostic.
- **OpenSpec layer**: Everything under the `openspec/` directory — acts as reference and orchestration only.
- **ToDoList**: The core application being developed in this repository (task management app).
- **SDD**: Spec-Driven Development (synonym for OpenSpec in this context).
- **Master artifact**: The canonical source of truth for a given piece of content (defined in `openspec/config.yaml`).

### Tool-Specific Terms
- **Continue.dev**: Primary AI coding assistant used in VSCode.
- **opsx**: Prefix for OpenSpec agent commands.

New terms should be added here and referenced from other files.
