# CODE_STYLE.md

## Code Style Guide

This project is a ToDoList application following modern industry standards with OpenSpec governance.

### General Rules
- Use consistent formatting (Prettier / ESLint / EditorConfig where applicable).
- All code must be readable and maintainable.
- Prefer explicit over implicit.
- Follow language-specific community standards.

### File Organization
- Industry-agnostic conventions → root files
- Tool-specific configurations → `.continue/`, `openspec/`, or standard dotfiles

### Documentation Style
- Use clear, concise language.
- Include examples when helpful.
- Keep artifacts modular and non-duplicated.

For detailed rules per language/technology, extend via `openspec/specs/`.
